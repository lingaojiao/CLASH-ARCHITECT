1. 解析节点 (Parse)

将您的代理节点分享链接（vmess/vless/ss/hy2 等）或现有的 YAML 配置文件源码粘贴到左上角的输入框中。

点击 ⚡ 解析节点，系统将自动识别协议并将其导入至 📦 节点池 中。

2. 可视化编排 (Arrange)

  • 创建组：在左侧输入组名并选择类型，点击“创建新分组”即可在右侧生成。
  • 拖拽操作：直接从节点池中将节点拖入下方分组。您也可以拖拽整个 📦 节点池 头部，将其中的所有节点一键导入某个组。
  • 组内嵌套：将已存在的 📂 文件夹图标（代表一个组）拖入另一个组内，实现复杂的层级转发逻辑。
  • 修改名称：双击任何一个节点标签，即可弹出重命名对话框。

3. 策略类型详解 (Policy Types)

  • select：手动选择。在客户端内由用户手动点击切换节点。
  • url-test：自动选择延迟最低。系统根据 间隔 设定的时间自动进行 HTTP 测速，并选择最快的节点。
  • fallback：故障转移。按列表顺序使用第一个健康节点，当主节点挂掉后自动按序顺延。
  • load-balance：负载均衡。根据 策略（如轮询或一致性哈希）分摊流量。

4. 载入YAML重构 (Reconstruct)

如果您手头有一个旧的 Clash 配置文件，直接粘贴其源码并点击 📥 载入YAML重构。

工具将自动逆向解析 proxies 和 proxy-groups，为您在右侧还原出原本的拖拽界面和层级关系，方便您进行微调。

5. 输出与维护

编排完成后，点击 🚀 生成 YAML 即可在底部查看源码。您可以使用复制或下载功能。

如果您想重新开始，点击左侧红色 🗑️ 清空页面 按钮。注意：这将重置所有节点和分组回到初始状态。
